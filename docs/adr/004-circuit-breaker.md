# ADR-004: 引入熔断器保护下游依赖

## 状态

已接受

## 日期

2025-02

## 上下文

系统依赖 PostgreSQL、Redis、Resend API 三个下游服务。任一下游故障可能导致请求堆积，进而拖垮整个服务（雪崩效应）。

## 决策

使用 sony/gobreaker 为每个下游依赖配置独立的熔断器：
- PostgreSQL: 5 次连续失败后熔断，30 秒后半开
- Redis: 5 次连续失败后熔断，15 秒半开
- Resend API: 3 次连续失败后熔断，60 秒半开

## 后果

**正面**
- 下游故障时快速失败，释放资源
- 半开状态自动探测恢复
- 防止雪崩效应

**负面**
- 熔断期间合法请求被拒绝
- 需要调优阈值（过低误触发，过高保护不足）
