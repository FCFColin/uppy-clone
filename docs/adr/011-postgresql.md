# ADR-011: PostgreSQL 作为持久化数据库

## 状态

已接受

## 日期

2026-06

## 上下文

多人游戏需要持久化用户、游戏会话、结果与管理配置。需在关系型与 NoSQL 之间选型。

## 决策

采用 **PostgreSQL 16**。

## 理由

- **ACID 事务**：游戏结果、Outbox 模式需要强一致性
- **JSONB**：房间状态序列化存储，兼顾 schema 灵活性与 SQL 查询
- **外键与 CHECK**：DB 层完整性约束（000003 迁移）
- **生态**：pgx v5、 golang-migrate、Cloud SQL / RDS 企业标准

## 备选方案

| 方案 | 放弃原因 |
|------|---------|
| SQLite/D1 | 水平扩展与并发写入受限 |
| MongoDB | 团队 SQL 熟练度；事务历史较弱 |
| CockroachDB | 当前规模过度；运维成本高（此否决仅适用于单区域阶段；多区域阶段见 ADR-015） |

## 权衡

- 写入扩展需读写分离 + 队列（ADR-007），非 PG 原生水平写
- 连接池需精心调优（`db_pool_*` 指标）
