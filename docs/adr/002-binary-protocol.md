# ADR-002: WebSocket 使用二进制协议而非 JSON

## 状态

已接受

## 日期

2025-01

## 上下文

游戏服务端以 **15Hz**（`protocol.TickRate = 15`，每 ~66.7ms 一帧）做权威 tick 与快照
广播；客户端在两帧之间做插值渲染以达到平滑视觉效果。每帧传输所有玩家位置、速度、
气球状态等数据。

## 决策

使用自定义二进制协议（protocol/encode.go, protocol/decode.go），而非 JSON。

## 后果

**正面**
- 带宽减少约 70%（无字段名、无文本编码开销）
- 编解码速度提升约 5x（无 JSON 解析）
- 固定偏移量读取，CPU 缓存友好

**负面**
- 不可读（调试需要 hex dump 工具）
- 需要维护编解码代码
- 版本兼容性需要手动管理
